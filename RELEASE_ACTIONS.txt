Release actions for v0.7.2

1) Commit and push
   git add -A
   git commit -m "chore(release): v0.7.2"
   git push origin HEAD

2) Tag and push
   git tag v0.7.2
   git push origin v0.7.2

3) Build
   .\.venv\Scripts\python.exe -m pip install --upgrade build
   .\.venv\Scripts\python.exe -m build --sdist --wheel --outdir dist

4) Upload
   .\.venv\Scripts\python.exe -m pip install --upgrade twine
   .\.venv\Scripts\python.exe -m twine upload dist/*

5) Draft GitHub release with tag v0.7.2 and paste `RELEASE_NOTES.md`.

Optional: Add `--basetemp=.pytest_tmp` to `pytest.ini` if Windows pytest cleanup fails in CI.
